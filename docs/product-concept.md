## 1. What It is

A reusable, client-side game engine that powers themed Wordle-style guessing games.
Each deployment is a standalone website with its own dataset and branding, but shares
the same core engine code.

The engine is the product. Each themed site (football players, celebrities, movies, 
historical figures, etc) is an instance of the engine configure with a different dataset.

## 2. Player Experience - Single Round of Play

### 2.1 Category Selection (Pre-game)

- The player arrives at a themed site (e.g. "Football Wordle").

- The screen displays category filters derived automatically from the dataset.

- The player can optionally narrow the word pool by selecting attribute values 
across categories.
    
    - Multiple values within one category act as **OR** filters (e.g. "Premier League" OR "La Liga")

    - Selections across different categories act as **AND** filters (e.g. "Striker" AND "Premier League").

    - Selecting nothing uses the full dataset

- A single word/entity in the dataset can have multiple values for one category (e.g. a player who played in multiple leagues).

- The player starts the round when ready.

### 2.2 The Grid

- The engine picks a random word from the filtered subset

- A grid appears with the correct number of letter slots for the selected word.

- For compound words or multi-word names, separators (visual dividers such as `/`) are pre-revealed in the grid. The player does not guess separators.

- The grid structure tells the player the word shape (e.g. a 9-letter first name and a 6-letter last name).

- There are always **6 rows** (attempts).

### 2.3 Guessing

- THe player types guesses using their physical keyboard or an on-screen keyboard.

- Guesses must match the exact letter count of the answer (excluding separators).

- Each submitted guess receives classic Wordle feedback per letter:

    - **Green:** correct letter in the correct position
    - **Yellow:** correct letter in the wrong position
    - **Gray:** letter not in the word.

- In V1, the player can type any text (no dataset validation). In a future version, guesses will be restricted to valid entries from the dataset.

### 2.4 Hints

- After each failed guess, a small `?` button appears on that row.

- The button is disabled until the attempt is confirmed as wrong, then becomes enabled.

- Clicking it reveals a hint about the target answer.

- Each word in the dataset has exactly **5 pre-authored hints**, ordered 1 through 5.

- Hint 1 is available after guess 1 fails, hint 2 after guess 2 fails, and so on up to hint 5 after guess 5.

- Hints are factual clues specific to the target word (e.g. birth year, nationality, achievements).

- Hints are authored as part of the dataset - they are not generated by the engine.

### 2.5 On-Screen Keyboard

- The game displays an on-screen keyboard(A-Z, Backspace, Enter).

- The keyboard serves as both an input method (tap/click) and a visual letter tracker.

- Letters are highlighted based on their best known state across all guesses in the current round:

    - Green if confirmed in the correct position
    - Yellow if confirmed in the word but not yet in the correct position
    - Gray if confirm absent

- No space key is needed - separators are auto-placed.

### 2.6 End of Round

- The player wins by guessing the word correctly within 6 attemps.

- On loss, the correct answer is revealed.

- Either way, the player can inmediately start a new round with the same or different category filters.

## 3 Engine vs Game Configuration

### 3.1 The Engine (Shared Code)

- Grid rendering with dynamic word lenght and pre-revealed separators.

- Letter-by-letter feedback logic (green / yellow / gray).

- Guess input handling with enforced length and auto-placed separators

- On-screen keyboard with letter state tracking.

- Category filter UI - derives categories and values from the dataset dynamically.

- Word selection - picks a random word matching the active filters.

- Hint system - manages 5 hints per word, reveals them progressively.

- Round lifecycle - start, guess, feedback, hint reveal, win/loss, new round

- Responsive layout that adapts to different word lengths

### 3.2 Game Configuration (Per Deployment)

- **Dataset:** list of words/entities, each with:
    - The word/name string.
    - Category attribute values (variable per theme)
    - Exactly 5 ordered hints.

- **Theme/branding:** site title, color scheme, optional logo.

- Configuration is provided as a **JSON file** loaded by the client at the startup.

- The engine makes **zero assumptions** about what categories exist - it discovers them from the data.

## 4. Technical Profile
 
| Aspect | Decision |
|---|---|
| Architecture | Fully client-side, no backend |
| Deployment | Static hosting (GitHub Pages, Netlify, Vercel, etc.) |
| Authentication | None |
| User accounts | None |
| Leaderboards | None |
| Daily word mechanic | None — unlimited play |
| Number of attempts | Always 6 |
| Guess validation (V1) | Free typing, length enforced |
| Guess validation (future) | Restricted to valid dataset entries |
| Configuration format | JSON file |
| Category derivation | Auto-derived from dataset |

## 5. Example Dataset Entry
 
```json
{
  "word": "Thierry Henry",
  "categories": {
    "position": ["Striker"],
    "country": ["France"],
    "league": ["Premier League", "La Liga"],
    "club": ["Arsenal", "Barcelona", "Monaco"],
    "era": ["1990s", "2000s"]
  },
  "hints": [
    "Born in 1977",
    "Born in Les Ulis, France",
    "All-time top scorer for the French national team",
    "Won the Premier League unbeaten",
    "World Cup winner in 1998"
  ]
}
```

## 6. Scope Boundaries

**In Scope (V1)**

- Fully functional game engine with all mechanics described above.

- One themed deployment as proof of concept.

- JSON-based configuration.

- Responsive web UI.

**Out of Scope(V1)**

- Backed/API

- User accounts and authentication.

- Leaderboards or statistics tracking.

- Daily word mechanic.

- Dataset validation against guesses (planned for future version)

- Admin UI for dataset management

## 7. Future Considerations (Not Committed)

- Guess validation againts the dataset or real words (reject invalid guesses - not real words).

- Sharing results (like Wordle's emoji grid).

- Multiple language support
